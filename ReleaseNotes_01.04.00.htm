Change Log

  • Added categories
    • Each event is in a category
    • Added CSS class based on category to calendar and default event listing templates
    • Categories can have a color associated
    • Default template and calendar display show categories' selected color
    • Categories can be assigned to a module instance, filtering what is displayed and editable through that instance
  • Enhanced Templating Engine
    • Supports various encodings of text values (HTML, JavaScript, CSS)
    • Supports sub-property access (i.e. Category-Color)
    • Supports attributes where only part of the attribute value is bound
  • Added custom permissions to define who can manage categories, events, templates, and responses
  • Updated the administrators' global navigation to be localizable, styleable, consistent, and more complete
  • Fixed bug where the numbering of responses was reset for each page of responses
  • Fixed bug where most of the Events-specific template tokens weren't available to the Details template
  • Fixed bug with displaying the Respond popup in DNN 5.4+
  • Fixed bug where Engage: Events was incompatible with many other Engage: modules
  • Fixed bug where multi-day events were cut-off in the calendar display
  • Fixed bug where events with recurrence rules that ended based on a date would only display in the listing display for the initial occurrence
  • Fixed bug where the resource files for built-in templates could not be accessed via the Language Editor
  • More details can be found on the Engage: Events Change Log

Known Issues

  • Some versions of modules from OnyakTech will conflict with Engage: Events. Starting in DNN 5.2, the Telerik assembly is shared with the core, so this issue should be resolved.
  • In DNN 5.2, the reference to the included Telerik assembly is incompatible with any other version. It is recommended to upgrade to DNN 5.2.2 or above to address this issue.
  • The entire range of a recurring event might not display on the Calendar view.  To increase performance, only 1,000 possible dates are considered to display.  This means that only about 33 years of a yearly recurrence will display, while close to the full 1,000 days of a daily recurrence will be shown.
  • If you have multiple events modules on the same page, paging through the events or responses lists on one module will cause the other module to change pages as well.  This can cause an issue if the other module has fewer pages than the module you are paging through, resulting in a blank module.
  • Line breaks entered in the Overview field of an event are not displayed; that is, all text displays on the same line, regardless of any line breaks entered.
